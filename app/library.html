Views

The app has three main views, accessible via the tabs at the top:

  • Characters: Your local library. Browse, search, filter, and manage all your character cards
  • Chats: Chat history browser. View all chat logs across characters, search by content, and see which AI model was used
  • Online: Browse and import characters from online providers (ChubAI, JanitorAI, CharacterTavern, Pygmalion, Wyvern)

On mobile, swipe left/right on the main screen to switch between views.

Display Modes

Character Library can run in two modes, configurable in SillyTavern's Extensions panel under Character Library:

  • New Tab (default): Opens in a separate browser tab.
  • Embedded Panel: Runs inside SillyTavern as an overlay panel.

Launcher

  • Show launcher dropdown on Characters button (default off): When off, a separate Character Library icon appears in SillyTavern's top bar and the Characters button behaves normally. When on, clicking the Characters button opens a dropdown to choose between SillyTavern's native character manager and Character Library.

Embedded Panel Settings

These options apply when Embedded Panel mode is selected:

  • Launch on startup: Automatically open the embedded panel when SillyTavern loads.
  • Show SillyTavern top bar: Keep SillyTavern's top navigation bar visible above the panel. When off, the panel takes the full viewport height and a "Back" button inside the panel returns you to your chat.
  • Exclusive panels: Opening the embedded panel closes any open SillyTavern drawers, and opening an ST drawer closes the panel. Prevents panels from overlapping.

Character Details

Click any character card to open their detail modal with tabs:

  • Details: Card fields (description, first message, personality, etc.), creator notes, and alternate greetings
  • Edit: Edit all card fields directly. Click Unlock Editing first to prevent accidental changes
  • Chats: All conversations with this character, with message counts and last-message preview. Resume any chat directly
  • Gallery: Images, GIFs, video, and audio. Download embedded media and provider gallery images
  • Related: Find similar characters in your library by shared tags, creator, and content overlap
  • Versions: Local snapshots and remote version history with diff preview (shown when history exists)
  • Info: Technical metadata: gallery ID, version UID, spec version, provider link details, extensions (enable in Settings)

Prev/Next Navigation

Cycle through your characters without returning to the grid:

  • Desktop: chevron buttons appear on the left and right sides of the open detail modal. Click to step to the previous or next character
  • Mobile: swipe left/right on the detail modal header (the avatar / name area). The tab-swipe inside the modal body is left untouched
  • Order follows your current sort and filter view: if you're sorted by Newest with a tag filter active, prev/next walks that filtered, sorted list
  • If you have unsaved edits in the Edit tab, you'll get a confirm prompt before swapping to a different character
  • Toggle the whole feature in Settings → Character Library → Card Grid & Browse → Prev/Next navigation in character details (on by default)

Context Menu

Right-click any character card (or long-press on mobile) for quick actions:

  • Open Character, Favorite/Unfavorite, Select for Batch
  • Link to Provider / Provider Info, Check for Updates
  • Version History, View Gallery, Export, Delete

When multiple characters are selected, the context menu shows bulk actions instead.

Mobile Tips

  • Tap the avatar in the detail modal header for a full-size quick-view
  • Swipe left/right on the detail modal header to step to the previous or next character in the current sort + filter view (toggleable in Settings, Card Grid & Browse)
  • Swipe left/right on character details to switch between tabs
  • Swipe left/right to cycle through alternate greetings
  • Long-press a character card to open the context menu (bottom sheet)
  • Swipe right on a card to toggle favorite, swipe left to open the context menu. The outer 12% per side and bottom 25% of each card are dead zones, so view-swipe and tab-swipe gestures starting near card edges aren't hijacked
  • Swipe left/right on the main screen to switch views (Characters ↔ Chats ↔ Online)
  • Tap-to-scroll long titles: tap a long character name in the detail modal header, or a long chat title in the chat preview header, to scroll through the full text. Tap again to reset
  • Mobile search overlay: opens at the bottom of the screen above the bottom nav when the keyboard is closed, and lifts above the keyboard when it opens, so the whole flow stays in the thumb zone
  • Android back button: closes the topmost open overlay (gallery viewer, confirm dialog, detail modal, etc.) before exiting the app

Search Filters

Use special search prefixes to filter your character library:

creator:name Filter by author/creator name
linked:yes / linked:no Filter characters linked/not linked to any provider
chub:yes / chub:no Filter by ChubAI link status
janny:yes / janny:no Filter by JanitorAI link status
ct:yes / ct:no Filter by CharacterTavern link status
pygmalion:yes / pygmalion:no Filter by Pygmalion link status
wyvern:yes / wyvern:no Filter by Wyvern link status
dc:yes / dc:no Filter by DataCat link status
favorite:yes / fav:no Filter by favorite status
version:text Filter by character_version field (version:none for empty)
gallery:id Filter by gallery_id (gallery:none for characters without one)
uid:id Filter by version_uid (uid:none for characters without one)
playlist:name Filter by playlist name (playlist:none for unassigned, playlist:any for in any playlist)

Combine prefixes with each other and with regular text: creator:john linked:yes dark elf finds linked characters by "john" matching "dark elf".

Tag Filtering

Click tags in the filter panel to cycle through states:

  • Green - Include: only show characters with this tag
  • Red - Exclude: hide characters with this tag
  • Gray - Neutral: no filter applied

Multiple tag filters combine: include tags use OR logic (match any), exclude tags are all applied.

Batch Tagging

Add or remove tags from multiple characters at once:

  • Desktop: Press Space or click the multi-select button in the toolbar to enter selection mode
  • Mobile: Long-press a card → Select for Batch, or open the More menu → Select Mode
  • Click characters to select/deselect, then click Edit Tags in the selection toolbar
  • The tag editor shows all tags across selected characters with counts; you can see which tags are shared by all vs. only some
  • Type new tags to add, or click existing tags to remove them

Filter Presets

Save your current filter configuration and reload it later with one click:

  • Click Presets in the Advanced Filters panel to open the dropdown
  • Type a name in the input and click Save to capture the current filter state (tags, sort, search, advanced filters, NSFW toggle, etc.)
  • Click any saved preset to load it. Use the input as a search box to filter the preset list when you have many
  • Hover a preset to rename or delete it
  • Set a Default Filter Preset in Settings (under Default Sort) to auto-load one preset every time the library opens
  • Presets are also available in the Chats tab for filtering chat history with the same UI

Useful for switching between focused workflows: a "sfw fantasy" preset, a "my tagged characters" preset, a "recently created" preset, etc.

Provider Linking

Link your local characters to their online source for update checking, version history, and gallery downloads. Supported providers:

  • ChubAI: Card updates, gallery downloads, full Git version history
  • JanitorAI: Card linking and updates
  • CharacterTavern: Card linking and updates
  • Pygmalion: Card linking, updates, and gallery downloads
  • Wyvern: Card linking, updates, and gallery downloads

How to link:

Click the Link indicator in the character detail sidebar, or right-click → Link to Provider. Search by name across all providers, or paste a direct URL.

Once linked:

  • Check for Updates: Compare local vs. remote card fields with a diff preview
  • Gallery Downloads: Download the character's provider gallery images
  • Version History: Browse published version history and restore any version
  • Quick Navigation: Jump to the provider page or preview in-app

Auto-Link

Bulk-link your entire library automatically:

  • Open from More Options (⋮) → Auto-Link Characters
  • Scans all unlinked characters by searching each provider's API by name
  • Results are categorized: Confident (strong match, auto-selected), Uncertain (multiple matches, you pick), and No Match
  • Review all matches before applying. Nothing is saved until you confirm
  • The scan is resumable: stop and resume later, or close and reopen without losing progress

Live status display while the scan runs:

  • The avatar and name of the character currently being scanned
  • A status line describing the current step (querying providers, comparing matches, etc.)
  • Progress counter (e.g. 147/892) and a progress fill bar showing scan completion
  • Running totals updated in real time: confident, need review, and no match counts
  • Once the scan finishes, the same counts double as tabs to filter the results list

Online Browsing

The Online tab lets you browse and import characters from providers:

  • Switch between providers using the dropdown selector
  • Search, sort, and filter by tags/categories. Each provider has its own options
  • Toggle NSFW content visibility per provider
  • Infinite scroll automatically loads more results as you scroll down (toggleable per provider in Settings)
  • Cards already in your library are marked with an "In Library" badge
  • Click a card to preview the full character details and gallery images, then import with one click
  • Set Collapse all browse preview sections by default in Settings → Character Details to start each preview with all sections collapsed except creator's notes (cleaner view for large cards)
  • Some providers support login for additional features (favorites, following lists)

Configure provider order and NSFW defaults in Settings → Online.

DataCat / JanitorAI

The DataCat provider combines multiple data sources to give you the best coverage for JanitorAI characters. Each sort order pulls from a different source:

DataCat sort orders (Recent, Fresh 24h/7d):

  • Characters come from DataCat's own database, which mirrors JanitorAI
  • Full character definitions are available (personality, scenario, first message, etc.)
  • Tags use DataCat's own category system
  • Creator names are always available
  • One-click import with complete card data
  • The search bar only accepts DataCat or JanitorAI URLs and creator UUIDs. No text search

JanitorAI (MeiliSearch) sort orders (Newest, Oldest, Most/Least Tokens, Relevant):

  • Searches JanitorAI's full-text search index directly
  • Covers all published characters on JanitorAI
  • The search bar supports text search (searches character names and descriptions) as well as JanitorAI URLs for direct lookup
  • Tags use JanitorAI's own tag system (Male, Female, Anime, etc.)
  • Creator names may be missing on grid cards (MeiliSearch doesn't always include them), but will appear in the preview modal if the character exists on DataCat
  • Characters not yet on DataCat can be extracted on demand

JanitorAI (Hampter) sort orders (Trending, Popular):

  • Pulls from JanitorAI's curated trending/popular feed
  • Only surfaces characters with significant engagement, not a complete index
  • The search bar supports text search (filters within the trending/popular pool only) as well as JanitorAI URLs
  • Tag filtering is not available in this mode
  • Creator names are always available

Cloudflare protection on Hampter: JanitorAI gates its trending/popular feed behind Cloudflare bot protection, so these sort orders only work when a FlareSolverr instance is configured in Settings → Online → DataCat (proxied through the cl-helper plugin). Without it, Hampter sorts will show a friendly notice explaining what to do; the other DataCat and JanitorAI sort orders are unaffected. Once configured, sessions are kept warm across tab switches so subsequent fetches skip the multi-second challenge.

Across all modes: Pasting a JanitorAI character URL into the search bar will look up the character on DataCat. If it's found, the preview opens directly. If not, an extraction panel appears letting you request the character from DataCat's cloud service.

Extraction: When a character only exists on JanitorAI and hasn't been indexed by DataCat yet, the Import button changes to Extract. This sends a request to DataCat's cloud service, which visits the JanitorAI page and retrieves the full character definition. You can also paste a JanitorAI URL directly into the search bar to look up and extract specific characters.

DataCat requires the cl-helper server plugin. Install it from Settings → Online → DataCat.

Re-extract on update checks: Toggle in Settings → Online → DataCat to force DataCat to pull fresh data from JanitorAI before each update check. Slower, but avoids comparisons against stale cached data on DataCat's side.

Version History

Track changes to your character cards with snapshots and diffs:

  • Auto-backup: A snapshot is saved automatically before every edit, restore, or card update, so you can always undo
  • Manual snapshots: Save the current state of any character at any time
  • Diff preview: Side-by-side comparison for every card field with added/removed highlighting
  • Remote versions: For linked characters, browse the full published version history and restore any version

How ChubAI remote versions work:

  • "Chub Page" entry: the current state on ChubAI, using the metadata API (with linked lorebooks resolved from the V4 Git API)
  • Git version history: each commit fetched from the V4 Git API (card.json), including the full merged lorebook

Configure max auto-backup count in Settings → General → Version History.

Card Editing

Edit all character card fields directly in the library:

  • Open a character → switch to the Edit tab → click Unlock Editing
  • All fields are locked by default to prevent accidental changes
  • Editable fields: name, description, personality, scenario, first message, alternate greetings, example messages, system prompt, post-history instructions, creator, creator notes, character version, tags, and the embedded lorebook
  • Use the expand buttons for full-screen editing of long text fields
  • Replace card image: with editing unlocked, click the large card image (or tap the avatar in the mobile header) to pick a new PNG. The pending image previews live; an X badge reverts before save. The auto-snapshot embeds the previous image, so version history can still show the old image after overwrite
  • A snapshot is auto-saved before each edit so you can always undo

Lorebooks & Card Updates

When a character card includes a lorebook (World Info), SillyTavern stores it in two places:

  • Embedded lorebook (inside the card): created at import time, not modified by SillyTavern's World Info editor
  • World Info file (in /worlds/): the live working copy used in chats. All your edits go here

These two copies are independent and diverge after import.

During a card update check:

  • The card's embedded lorebook is compared against the creator's latest version on the provider
  • New, modified, and local-only entries are shown in the diff
  • Your World Info file is never modified by the update checker

When you apply a lorebook update:

  • The card's embedded lorebook is replaced with the latest version
  • Local-only entries (in your card but not on the provider) are removed from the embedded copy
  • Your World Info file is not touched

Safety check for local-only entries:

  • Safe: also exists in your World Info file
  • Warning: not found in your World Info file, will be permanently lost

TL;DR: Updates only change the card's embedded data. Your World Info file (the copy that matters for chats) is never modified.

Display Name Override

For characters linked to a provider, you can choose whether the library shows the card's own name or the provider listing name (the title used on the provider's page). The card data is never modified; this is purely a display preference.

  • Open a character → click the name type toggle in the details header to switch between card name and listing name for that character
  • Set the global default in Settings → General → Display name source (Card name or Listing name)
  • Find characters with a listing name available using the Advanced Filters panel
  • SillyTavern integration: When enabled in Settings, the resolved display name (whichever you've chosen) is also pushed into ST's chat header, character panel, and Character Management sidepane

Useful when a creator's listing title differs from the card's internal name field, e.g. "Ada (Cyberpunk Bartender)" on the listing vs. just "Ada" on the card.

Duplicate Detection

Find potential duplicates in your library:

  • Open from More Options (⋮) → Find Duplicates
  • Compares character name, creator, description, tags, and lorebook entries
  • Shows a similarity score (0-100%) for each match
  • Review matches and delete unwanted duplicates

Adjust the minimum score threshold in Settings to control sensitivity.

Grid Card Thumbnails

Cut decode cost and bandwidth on the characters grid by serving smaller, pre-resized images for the grid cards. Detail modal, gallery, and full-screen viewer always keep full resolution.

  • Enable in Settings → Character Library → Grid Card Thumbnails → Use thumbnails in characters view
  • Applies to mobile-sized viewports only by default. Toggle "Also use on desktop" to extend coverage
  • Two backends:
    • cl-helper (recommended): resizes via jimp and caches each thumbnail on disk at the size you pick (384 / 512 / 640 / 768px wide). 512 is a good baseline. Requires the cl-helper plugin installed
    • ST's built-in /thumbnail: fixed at 96x144, which looks blurry on high-DPR phones. Used automatically if cl-helper isn't available or you turn the cl-helper option off
  • Cache management (cl-helper backend, in the same settings group):
    • Populate at current size: pre-generates a thumbnail for every character at the size selected above. Already-cached entries are skipped. Useful after a bulk import or after changing the size
    • Purge cache: deletes every cached thumbnail (any size). Next scroll regenerates on demand
    • Live cache stats (count + on-disk size) update under the buttons

Biggest win: mobile decode cost drops sharply on fast-scroll, and large libraries on slow connections feel snappier. On desktop with plenty of decode budget, the win is mostly bandwidth.

Gallery Viewer

Full-screen media viewer for character gallery images, GIFs, and videos:

  • Open from right-click → View Gallery, or from the character detail modal
  • Navigate with arrow keys (← →), click left/right halves of the image, or use the prev/next buttons
  • Scroll wheel to zoom (up to 5×), drag to pan when zoomed, click or press 0 to reset zoom
  • Thumbnail strip at the bottom for quick jumping between images
  • Supports images, GIFs, and video (mp4/webm). Press Escape to close

Media Localization

Download remote media (images, audio, video) referenced in character cards for offline access:

  • Scans card text fields for remote URLs and downloads each file to the character's gallery folder
  • Also downloads gallery images from linked providers
  • Deduplication by file hash; won't re-download files you already have
  • Some hosts require SillyTavern's CORS Proxy to be enabled (User Settings → Network)

Enable "Replace remote media URLs" in Settings to auto-swap URLs when viewing. Use Bulk Localization in Settings → Media to process your entire library at once.

Gallery Sync

When Unique Gallery Folders is enabled (Settings → Gallery Folders), each character gets their own gallery folder even if they share names.

  • Gallery Sync audits the integrity of folder mappings on startup
  • Identifies: missing gallery IDs, broken folder mappings, orphaned mappings for deleted characters
  • View the audit status from More Options (⋮) → Gallery Sync Status
  • Can auto-repair missing IDs and mappings

External Gallery Extractors

Some character cards can link to galleries on external image hosts instead of embedding direct image URLs. Character Library walks these links, resolves the underlying images, and downloads them into the character's gallery folder.

Supported hosts:

  • Civitai: posts and image collections (public out-of-the-box; private/hidden posts require an API key)
  • Imgchest: public and password-protected posts (passwords are auto-detected from the card's creator notes / description via regex; password-protected posts require cl-helper)
  • Mega.nz: public folder shares with multi-pass folder traversal
  • Imgur, PostImg, ImgBB, Imgbox: albums and individual images
  • Catbox: direct file URLs
  • Dropbox: shared folders and individual file links
  • Google Drive: shared folders and individual files

Extractors run automatically when you trigger Download Embedded Media from the Gallery tab or use Bulk Localization. No configuration is required for public content.

Civitai API key: Required only for private or hidden Civitai posts. Configure in Settings → Online → Civitai. Generate at civitai.com/user/account. The key is stored persistently and pushed to the cl-helper plugin at startup, so it stays server-side at request time.

AI Recommender

Get AI-powered character recommendations based on a natural-language prompt:

  • Open from More Options (⋮) → Card Recommender
  • Describe what you're looking for (e.g. "cozy fantasy characters", "dark horror scenarios")
  • Samples characters from your library and sends their metadata to an LLM
  • Works with your active SillyTavern Chat Completion source, or a custom OpenAI-compatible API
  • Batch Mode for wider library coverage: splits the pool across multiple LLM calls
  • Filter the sample pool by favorites, chat history, date range, and tags
  • Click any result card to expand its reasoning, or tap it to open the character
  • Add individual characters to a playlist from the result card, or use Add all to playlist to save every recommendation at once
  • Reroll to re-run with the same prompt and settings; previously picked characters are excluded so you get fresh suggestions

The recommender only sends character names, tags, creator, and taglines, not conversation history or full card text.

Importing Characters

  • From URL: Paste one or more character URLs (one per line) from any supported provider

Chat History Browser

The Chats tab lets you browse all chat histories across all characters:

  • Two view modes: flat grid (all chats as cards) or grouped by character
  • Sort by date, character name, message count, chat length, or most active character
  • Each chat card shows character name, last message preview, message count, date, and AI model badge
  • Click to preview the full conversation with rendered messages
  • Open chats directly in SillyTavern or delete them

Overview

Create new characters from scratch or edit existing ones with AI assistance. Open via ⋮ menu → Create a Character.

Each card field (description, personality, scenario, first message, etc.) has a wand icon button that opens the AI Studio panel for that field.

AI Studio

The AI Studio is a conversational assistant for generating and refining card fields:

  • Multi-turn chat: Ask for rewrites, adjustments, or entirely new content. The full conversation history is sent with each request so the LLM has context
  • Suggestion chips: Quick-start prompts tailored to each field (e.g. "Write a mysterious backstory", "Make them sarcastic and witty"). Click one to generate immediately
  • Word target: Set a target word count for generations. The LLM will aim to match it
  • Apply to card: When satisfied with the result, click Apply to Card to write it into the card field
  • Undo / Redo: History stack for the current studio session. Use the toolbar buttons or Ctrl+Z / Ctrl+Y (when not focused in a text input). Opening a different field starts a fresh history

Brainstorm Mode

When editing the Description field, a lightbulb icon appears in the AI Studio toolbar. Click it to enter Brainstorm Mode:

  • The content editor hides and the conversation area expands to fill the panel
  • The LLM becomes a collaborative design partner: it asks questions, suggests ideas, and helps you develop your character concept through back-and-forth dialogue
  • Suggestion chips offer creative starting points like "a mysterious character with a hidden past" or "help me figure out what motivates this character"
  • When you're happy with the concept, click Finalize. The LLM synthesizes the entire conversation into a polished character description and writes it into the content area
  • Click the lightbulb again to exit brainstorm mode and return to normal editing

Brainstorm mode is great when you have a vague idea but aren't sure how to put it into words yet. Let the LLM interview you instead of staring at a blank page.

Highlight Revision

Select (highlight) any portion of text in the AI Studio content area to enter revision mode:

  • The selection is locked and visually highlighted with a colored overlay
  • A "Revising selection" badge appears above the input row
  • Your next prompt targets only the highlighted section. The LLM rewrites just that segment and splices it back into the surrounding text
  • Click the × on the badge to clear the selection and return to full-field mode

This is useful for tweaking a single paragraph without regenerating the entire field.

Custom System Prompts & Presets

Click the gear icon in the AI Studio toolbar to open field-specific settings:

  • Custom system prompt: Override the built-in instruction for that field with your own. The placeholder shows the default prompt for reference
  • Context checkboxes: Toggle which other card fields are included as context when generating. For example, uncheck "Personality" when generating a description if you want a fresh take
  • Save as preset: Save your custom prompt as a named preset. Load or delete saved presets from the dropdown

Presets are saved per field and persist across sessions. A dot indicator appears on the gear icon when a field has active overrides (custom prompt or disabled context fields).

Connection Profiles

If you have multiple Chat Completion sources configured in SillyTavern, a Connection Profile dropdown appears in the sidebar under AI Assist. Select which profile to use for AI generation, letting you use a different model than your chat model.

Import from Library

Click Import from Library to load an existing character into the creator form. This populates every field including tags, avatar, alternate greetings, and lorebook. You can then modify and save as a new character or overwrite the original.

Save Modes

  • Create Character: Saves as a brand-new character in your library
  • Save as Existing: Appears after importing from library. Opens a side-by-side diff review showing exactly what changed before overwriting the original card. An automatic snapshot is saved first so you can always undo
